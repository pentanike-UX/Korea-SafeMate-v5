# 01 — 제품 PRD (코드 기준)

> 과거 기획서 원문은 복원 불가. 아래는 **현재 구현**과 **git 메시지**로 추정 가능한 범위만 기술.

## 문제 정의

- 한국 여행에서 **같은 도시·권역 안**에서 스팟 순서·이동·체류 시간을 한 번에 짜기 어렵다.
- 출발/귀경 왕복은 다루지 않고 **현지 로컬 동선**에 집중한다 (`/api/v5/chat` 시스템 프롬프트 기준).

## 핵심 사용자

- 한국 여행자(국내·해외), 모바일·데스크톱.
- 비로그인(게스트) 사용 가능하나 저장은 로그인 권장(UI·문구로 안내).

## 핵심 기능 (구현 기준)

1. **채팅 `/chat`**  
   - Gather: 자연어 → 칩 + `readyToGenerateRoute`.  
   - Plan: 확정 슬롯 → `travelPlan` JSON (스팟 2–12, lat/lng nullable).
2. **하이브리드 입력**  
   - 8슬롯 칩(지역·구역·일정·인원·교통·분위기·음식 등) + 자유 입력 세그먼트(상호 배타).
3. **동선 카드**  
   - 저장, 지도 모달, 게스트 시 로그인 유도 배너.
4. **지도 모달**  
   - MapLibre, 경로 geometry, 스팟별 Tour/위키 보강.
5. **사이드바**  
   - 나의 플랜, 대화 히스토리, 유저바(로딩 스켈레톤).
6. **로그인**  
   - Google OAuth (`/login`), i18n 카피.
7. **홈 `/`**  
   - 랜딩·CTA → 채팅 (세부는 `src/app/[locale]/(public)/page.tsx`).

## 비기능

- Wayly 무료 한도·Stripe Wayly+: `env.example`, 요금 모달 코드 참고.
- Tour API 일일 할당량: 클라이언트에서 동시성 제한(`use-plan-tour-enrichment` 등).

## 명시적 비범위 (코드/커밋에서 유추)

- v3 대비 가디언 매칭·안전 동행 제거 방향 — `docs/wayly-v5-ia-functional-spec.md` 서술와 일치 **(문서 근거)**.
- 현재 `src/app` 기준 **라우트 파일은 홈·채팅·로그인 3개만** 존재 — 탐색/가디언 등은 제거되었거나 다른 진입 없음 **(코드 근거)**.
