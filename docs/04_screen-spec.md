# 04 — 화면 스펙 (코드 기준)

> 픽셀 단위 디자인 시스템은 별도 Figma 없음. 아래는 **구현 관찰** + **inferred** 구분.

## 공통

- 다크/라이트: 테마 토글(사이드바 더보기 등).
- CSS 변수: `--brand-*`, `--bg-*`, `--text-*`, `--border-*` (globals / 컴포넌트 클래스).

## 홈 `/`

- **목적**: 서비스 소개 후 채팅 진입.
- **구현 파일**: `src/app/[locale]/(public)/page.tsx` 및 연결 컴포넌트.

## 채팅 `/chat`

### 레이아웃

- **데스크톱 (lg+)**: 좌 사이드바 + 메인 메시지 + (분할 시) 우측 타임라인·지도 프리뷰.
- **태블릿/모바일**: 드로어·풀스크린 오버레이·하단 입력 독 (커밋 히스토리상 반복 개선).

### 빈 상태 (EmptyState)

- Wayly 마크 + 안내 문구 + **「여행 조건 입력하기」** 등 CTA → 하단 composer 스크롤 (merge `2dc8fc4` 이후).
- 중복 하이브리드 카드·템플릿 그리드는 제거됨.

### 입력

- **간편 선택**: 8칩, 지역·일정 필수(`*`), 미충족 시 전송 토스트.
- **자유 입력**: 첫 메시지에 지역·일정 힌트 없으면 차단 + 토스트 (동 커밋군).
- **일정**: 당일/N박 프리셋 중심; 캘린더 직접 선택 UI는 제거 **(최근 코드)**.

### 메시지·동선 카드

- 어시스턴트: 칩 카드, `travelPlan` 카드, 저장/지도, 게스트 저장 유도.

### 모달

- 지도: `V5PlanMapModal` — 리스트, 상세 패널, 전체화면, 경로 로딩.
